Features

Everything you need to
understand what happened

From detection to replay to clip — Sim Steward handles the whole pipeline so you can focus on driving.

📡

Real-time incident detection

Reads iRacing session YAML at 60 Hz to detect 1x (off-track), 2x (spin/wall), and 4x (heavy contact) incidents the moment they're issued — for every driver on track.

🎬

Frame-accurate replay control

Dashboard replay panel lets you seek directly to any incident frame, step forward/back, change speed (including −1× rewind), and get instant visual feedback on replay state.

🎥

Automatic OBS clip recording

A lightweight Node.js bridge connects Sim Steward to OBS WebSocket 5. Each incident push auto-seeks iRacing to the exact frame and starts recording — clips named by session and frame.

🏁

Session summary at checkered flag

The moment the checkered flag drops, the plugin broadcasts a full session summary — complete results table plus the plugin's own incident feed for cross-validation.

🔭

Incident cause inference

At incident time, Sim Steward cross-references car proximity to infer the likely cause: contact, wall strike, or off-track excursion — displayed on every incident card.

📊

Grafana Loki observability

Structured logs pushed to Grafana Loki (cloud or local Docker). Four pre-built dashboards: Command Audit, Incident Timeline, Plugin Health, and Session Overview.

🌐

LAN-accessible dashboard

The HTML/JS dashboard is served by SimHub and accessible from any device on your LAN — second monitor, tablet, co-driver laptop. No install required on client devices.

⚡

Real-time WebSocket push

Plugin state and incident events are pushed over a Fleck WebSocket, not polled. The dashboard reflects changes within milliseconds — no page refresh needed.

🛡️

Deduplication & replay safety

Incidents are deduplicated by (CarIdx, ReplayFrameNum) so seek-backward and re-scans never create phantom duplicates.

How it works

From lap to clip in seconds

Sim Steward runs silently inside SimHub and handles the full incident pipeline without interrupting your session.

1

Install the SimHub plugin

Drop SimSteward.Plugin.dll into your SimHub plugins folder and restart. The plugin registers automatically — no XML config required.

2

Start iRacing as usual

Sim Steward detects your session automatically. The plugin begins reading YAML data at ~60 Hz and populating the incident feed the moment the race director issues a penalty.

3

Open the dashboard on any device

Navigate to http://<your-pc-ip>/Web/sim-steward-dash/ on your second monitor or tablet. The live incident feed updates in real-time over WebSocket.

4

Click any incident to seek instantly

Tap an incident card to send a SelectIncidentAndSeek command. iRacing's replay jumps to the exact frame and Sim Steward enters the incident review state automatically.

5

OBS records the clip (optional)

Run the Node.js OBS bridge once. Every new incident event triggers an automatic replay seek and StartRecord command — clips are named by session and frame number.

OBS Integration

Every incident,
automatically clipped

The OBS bridge connects Sim Steward's WebSocket to OBS WebSocket 5. Each incident triggers an exact-frame replay seek and starts recording before you've even reacted.

  • ✓
    Listens to incidentEvents WebSocket push
  • ✓
    Sends ReplaySeekFrame to jump to the exact incident frame
  • ✓
    Calls StartRecord in OBS automatically
  • ✓
    Clips named {sessionId}_inc_{frame}.mkv for easy cataloging
  • ✓
    Zero config for most setups — just npm install && npm start

Grafana Loki

Full session
observability

Every incident, command, and anomaly is pushed as structured JSON to Grafana Loki — cloud or local Docker. Four provisioned dashboards surface the data you care about.

  • ✓
    Incident Timeline — every event plotted by frame and driver
  • ✓
    Command Audit — full history of replay and seek commands
  • ✓
    Plugin Health — latency, errors, and reconnection events
  • ✓
    Session Overview — AI-friendly session digest for post-analysis
